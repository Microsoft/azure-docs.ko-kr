# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>Azure NetApp Files 리소스 공급자 오류 문제 해결
이 문서에서는 일반적인 Azure NetApp 파일 리소스 공급자 오류, 해당 원인, 솔루션 및 해결 방법을 설명 합니다. 

<a name="error_01"></a>***Azure Key Vault 구성 되지 않았습니다.***   
Azure Key Vault는 기본 API에 액세스 하기 위해 필요한 자격 증명을 저장 합니다. 이 오류는 Azure Key Vault 기본 API에 액세스 하기 위한 전체 자격 증명을 받지 나타냅니다.

* 원인  
Azure Key Vault에 올바른 자격 증명을 받지 또는 자격 증명이 완전 하지 않습니다.  

* 해결 방법   
Azure NetApp 파일 서비스는 Azure Key Vault를 사용합니다. Azure Key Vault는 Azure Active Directory에서 토큰을 사용 하 여 인증 합니다. 따라서 응용 프로그램의 소유자는 Azure Active Directory에서 응용 프로그램을 등록 해야 합니다.

* 해결 방법   
없음.  NetApp Azure Files를 사용 하 여에 대 한 azure Key Vault 올바르게 설정 해야 합니다.  

<a name="error_02"></a>***토큰 생성을 변경할 수 없습니다.***   
이 오류는 볼륨에 만들어진 후 생성 토큰을 변경 하려고 할 때 발생 합니다.
볼륨 생성 되 고 나중에 변경할 수 없는 경우 만드는 토큰을 설정 되어야 합니다.

* 원인   
지원 되는 작업이 없는 볼륨에 만들어진 후 생성 토큰을 변경 하려고 합니다.

* 해결 방법   
볼륨을 만든 후에 오류 메시지를 해제 하려면 요청에서 매개 변수를 제거 하는 것이 좋습니다.

* 해결 방법   
만들 토큰을 변경 해야 할 경우 새 생성 토큰을 사용 하 여 새 볼륨을 만들 수 있으며 다음 새 볼륨으로 데이터를 마이그레이션합니다.


<a name="error_03"></a>***토큰 생성 최소 16 자 여야 합니다.***   
이 오류는 생성 토큰 길이 요구 사항에 맞지 않을 때 발생 합니다. 만들 토큰의 길이 16 자 이상 이어야 합니다.

* 원인   
만들 토큰 길이 요구 사항에 맞지 않습니다.  API를 사용 하 여 볼륨을 만들면 생성 토큰은 필수입니다. 포털을 사용 하는 경우 토큰을 자동으로 생성할 수 있습니다.

* 해결 방법   
만들 토큰의 길이 늘립니다. 예를 들어, 시작 또는 생성 토큰의 끝에 다른 단어를 추가할 수 있습니다.

* 해결 방법   
최소한 생성 토큰의 길이 무시할 수 없습니다.  만들 토큰 길이 높이기 위해 접두사 또는 접미사를 사용할 수 있습니다.


<a name="error_04"></a>***Azure NetApp 파일에서 찾을 수 없는 볼륨을 삭제 하는 데 오류가 발생 했습니다.***   
리소스의 내부 레지스트리 동기화 되지 않았습니다.이 오류가 발생 했습니다.

* 원인   
볼륨 삭제 된 후 일정 시간 동안 포털에 표시 유지 될 수 있습니다. API를 사용 하 여 볼륨을 삭제 하는 경우 가능 볼륨 올바르게 지정 하지 않았습니다. 오류가 오래 된 브라우저 캐시에도 발생할 수 있습니다.

* 해결 방법   
포털을 사용 하는 경우 캐시를 지우기 브라우저입니다. 10 분 마다 새로 고쳐지는 내부 캐시 이기도 합니다.  캐시를 지우려면 다시 시도할 수 있습니다.  10 분 후 문제가 지속 되 면 지원 티켓을 만들 수 있습니다.

* 해결 방법   
그동안 다른 볼륨을 사용 하 고 기존을 무시 합니다.


<a name="error_05"></a>***Azure NetApp 파일에 있는 새 볼륨을 삽입 하는 데 오류가 발생 했습니다.***   
리소스의 내부 레지스트리는 동기화 되지 않으므로이 오류가 발생 합니다.

* 원인   
볼륨 삭제 된 후 일정 시간 동안 포털에 표시 된 남아 있을 수 있습니다. API를 사용 하 여 볼륨을 삭제 하는 경우 가능 볼륨 올바르게 지정 하지 않았습니다.

* 해결 방법   
포털을 사용 하는 경우 볼륨 이미 생성 되었습니다.  볼륨에 대 한 연결이 자동으로 나타납니다. 문제가 지속 되 면 지원 티켓을 만들 수 있습니다.

* 해결 방법   
다른 이름 및 다른 만들기 토큰을 사용 하 여 볼륨을 만들 수 있습니다.


<a name="error_06"></a>***파일 경로 이름에는 문자, 숫자 및 하이픈만 포함 될 수 있습니다 (""-"")만 합니다.***   
파일 경로는 지원 되지 않는 문자를 포함 하는 경우이 오류가 발생 예를 들어, 마침표 ("."), 쉼표 (","), 밑줄 ("\_"), 또는 달러 기호 ("$").

* 원인   
파일 경로는 지원 되지 않는 문자가 포함 되어 있습니다. 예를 들어, 마침표 ("."), 쉼표 (","), 밑줄 ("\_"), 또는 달러 기호 ("$").

* 해결 방법   
알파벳 문자, 숫자 또는 하이픈 하지 않은 문자 제거 ("-")에서 파일 경로 입력 합니다.

* 해결 방법   
하이픈으로 밑줄을 대체 수도 있고 공간 대신 대/소문자를 사용 하 여 (예: "새 볼륨" 대신 "NewVolume"를 사용 하 여) 새 단어의 시작을 나타냅니다.


<a name="error_07"></a>***볼륨 ID는 변경할 수 없습니다.***   
볼륨 ID를 변경 하려고 할 때이 오류가 발생  볼륨 ID를 변경 하는 것은 지원 되는 작업이 아닙니다.

* 원인   
파일 시스템의 ID에는 볼륨을 만들 때 설정 됩니다. 볼륨 ID는 나중에 변경할 수 없습니다.

* 해결 방법   
없음.

* 해결 방법   
없음.  볼륨 ID는 볼륨 생성 되 고 나중에 변경할 수 없는 경우에 생성 됩니다.


<a name="error_08"></a>***잘못 된 값 '{0}'에 대해 받은 {1}합니다.***   
이 메시지는 RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3, 및 Nfsv4 필드에 오류를 나타냅니다.

* 원인   
다음 필드 중 하나 이상에 대 한 입력된 유효성 검사 요청이 실패 했습니다. RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3, 및 Nfsv4 합니다.

* 해결 방법   
명령줄에 모든 필수 및 충돌 하지 않는 매개 변수를 설정 해야 합니다. 예를 들어 동시 UnixReadOnly와 UnixReadWrite 매개 변수를 설정할 수 없습니다.

* 해결 방법   
해결 방법 섹션을 참조 하세요.

이 오류에 필요한 특성이 다음 매개 변수 중 하나 이상에 대 한 요청에서 누락 된 것을 나타냅니다. RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3, 및 Nfsv4 합니다.

* 원인   
다음 필드 중 하나 이상에 대 한 입력된 유효성 검사 요청이 실패 했습니다. RuleIndex, AllowedClients, UnixReadOnly, UnixReadWrite, Nfsv3, 및 Nfsv4 합니다.

* 해결 방법   
명령줄에 모든 필수 및 충돌 하지 않는 매개 변수를 설정 해야 합니다. 예를 들어 동시 UnixReadOnly와 UnixReadWrite 매개 변수를 설정할 수 없습니다 있습니다.

* 해결 방법   
해결 방법 섹션을 참조 하세요.  


<a name="error_10"></a> ***{0} 이미 사용 중입니다.***   
이 오류 리소스에 대 한 이름이 이미 사용 되는 것을 나타냅니다.

* 원인   
기존 볼륨으로 동일한 이름의 볼륨을 만들 하려고 합니다.

* 해결 방법   
볼륨을 만들 때 고유한 이름을 사용 합니다.

* 해결 방법

<a name="error_11"></a> ***{0} 너무 짧습니다.***

볼륨 이름을 너무 짧습니다.

* 해결 방법   
볼륨 이름 길이 늘립니다.

볼륨 이름에는 공통 접두사 또는 접미사를 추가할 수 있습니다.


<a name="error_12"></a>***Azure NetApp 파일 API를 연결할 수 없습니다.***   
Azure API는 Azure NetApp 파일 API 볼륨 관리를 사용 합니다.  이 오류는 API 연결을 사용 하 여 문제를 나타냅니다.

* 원인   
기본 API 응답 하지 않는 내부 오류가 발생 합니다. 이 오류는 일시적일 수 있습니다.

* 해결 방법   
문제가 일시적인 가능성이 있습니다.  요청은 일정 시간이 지난 후 성공 해야 합니다.

* 해결 방법   
없음. 기본 API는 볼륨을 관리 하는 데 필수적입니다.  


<a name="error_13"></a>***구독에 대 한 자격 증명 없이 찾을 '{0}'.***   
이 오류는 제공된 된 자격 증명 잘못 기능은 구독에서 올바르게 설정 되지 않은 나타냅니다.

* 원인   
볼륨 관리에 대 한 서비스에 대 한 액세스를 방지 하는 자격 증명이 잘못 되었거나 잘못 설정 합니다.

* 해결 방법   
자격 증명 집합과 명령줄에서 올바르게 입력 했는지 확인 하십시오.

* 해결 방법   
없음.  자격 증명을 올바르게 설정 하는 것은 필수적 NetApp Azure Files를 사용 하 여입니다.  


<a name="error_14"></a>***Id가 없습니다. 작업 결과 '{0}'.***

* 해결 방법   
몇 분 기다렸다가 문제가 지속 되 면 확인 합니다.


<a name="error_15"></a>***작업 '{0}' 지원 되지 않습니다.***

변경할 수 없는 (예: 볼륨 ID) 매개 변수를 수정 하려고 했습니다.

<a name="error_18"></a>***요청 된 {0} 찾을 수 없습니다.***

철자가 잘못 된 리소스 이름에는 이미 삭제 되었거나는 존재 하지 않는 리소스 (예를 들어, 볼륨 또는 스냅숏)를 참조 하려고 합니다.

* 해결 방법   
올바르게 참조 되는지 확인 하려면 맞춤법 오류에 대 한 요청을 확인 합니다.

* 해결 방법   
해결 방법 섹션을 참조 하세요.

잘못 된 자격 증명이 올바르게 구독에 설정 하지에 액세스 하지 못하도록 볼륨을 관리 하기 위한 서비스 또는 합니다.

* 해결 방법   
자격 증명 집합과 명령줄에서 올바르게 입력 했는지 확인 하십시오.

없음.  올바르게 집합 자격 증명이 Azure NetApp 파일을 사용 하는 데 필수적입니다.

<a name="error_20"></a>***알 수 없는 Azure NetApp 파일 오류가 발생 했습니다.***   
Azure API는 Azure NetApp 파일 API 볼륨 관리를 사용 합니다. 이 오류는 API에 대 한 통신 문제를 나타냅니다.

기본 API는 알 수 없는 오류를 전송 합니다.  이 오류는 일시적일 수 있습니다.

문제는 일시적 이며 수 및 요청은 일정 시간이 지난 후 성공 해야 합니다. 문제가 지속 되 면 조사 하는 문제가 있음을 나타내기 위해 지원 티켓을 만듭니다.

* 해결 방법   
없음.  기본 API는 볼륨을 관리 하는 데 필수적입니다.

<a name="error_21"></a>***알 수 없는 속성에 대해 수신 하는 값 '{0}'.***   
볼륨을 스냅숏 또는 탑재 대상을 같은 리소스에 대 한 존재 하지 않는 속성을 제공 하는 경우이 오류가 발생 합니다.

* 원인   
요청에는 각 리소스를 사용 하 여 사용할 수 있는 속성 집합이 있습니다.  요청에 존재 하지 않는 모든 속성을 포함할 수 없습니다.

* 해결 방법   
모든 속성 이름의 철자가 올바른지와 구독 및 리소스에 대 한 속성을 사용할 수 있는지 확인 합니다.

* 해결 방법   
오류의 원인이 되는 속성을 제거 하는 요청에 정의 된 속성의 수를 줄입니다.


<a name="error_22"></a>***이 리소스 종류에 대 한 업데이트 작업이 지원 되지 않습니다.***   
볼륨만 업데이트할 수 있습니다. 이 오류 발생, 지원 되지 않는 업데이트 작업을 수행 하려고 할 때 예를 들어 스냅숏을 업데이트 합니다.

* 원인   
업데이트 하려는 리소스 업데이트 작업을 지원 하지 않습니다.  볼륨만 수정 해당 속성을 가질 수 있습니다.

* 해결 방법   
없음.  업데이트 하려는 리소스 업데이트 작업을 지원 하지 않습니다. 따라서 변경할 수 없습니다.

* 해결 방법   
볼륨에 대 한 곳에서 업데이트를 사용 하 여 새 리소스를 만들고 데이터를 마이그레이션하십시오.


<a name="error_23"></a>***항목 수: {0} 개체에 대 한: {1} 최소-최대 범위를 벗어났습니다.***   
이 오류는 내보내기 정책 규칙의 최소값 또는 최대값 범위 요구 사항을 충족 하지 않는 경우에 발생 합니다.  내보내기 정책을 정의 하는 경우 최소한 하나의 내보내기만 정책 규칙 및 최대 5 개의 내보내기 정책 규칙 있어야 합니다.

* 원인   
정의한 내보내기 정책을 필요한 범위를 충족 하지 않습니다.  

* 해결 방법   
인덱스에 이미 사용 하지 않는지 확인 범위는 1에서 5에서입니다.

* 해결 방법   
볼륨에 내보내기 정책을 사용 하는 필수적이 지 않습니다. 따라서 내보내기 정책 규칙이 있이 필요가 없습니다 경우에 완전 하 게 내보내기 정책의 생략할 수 있습니다.


<a name="error_24"></a>***중복 된 개체에 대 한 값 오류 {0}합니다.***   
이 오류는 내보내기 정책을 고유 인덱스를 사용 하 여 정의 되지 않은 경우에 발생 합니다.  내보내기 정책을 정의할 때 모든 내보내기 정책 규칙 1과 5 사이의 고유 인덱스가 있어야 합니다.

* 원인   
정의 된 내보내기 정책을 내보낼 정책 규칙에 대 한 요구 사항을 충족 하지 않습니다. 최소한 하나의 내보내기만 정책 규칙 및 최대 5 개의 내보내기 정책 규칙 있어야 합니다.  

* 해결 방법   
인덱스가 이미 사용 되지 않습니다 및 범위는 1에서 5는 있는지 확인 합니다.

* 해결 방법   
설정 하려고 하는 규칙에 대 한 다른 인덱스를 사용 합니다.
