# <a name="quickstart-add-a-simulated-device-to-your-iot-central-application"></a>빠른 시작: IoT Central 애플리케이션에 시뮬레이션된 디바이스 추가

디바이스 템플릿은 IoT Central 애플리케이션에 연결하는 디바이스의 기능을 정의합니다. 기능에는 디바이스에서 보내는 원격 분석, 디바이스 속성 및 디바이스에서 응답하는 명령이 포함됩니다. 디바이스 템플릿을 사용하여 실제 디바이스와 시뮬레이션된 디바이스를 모두 애플리케이션에 추가할 수 있습니다. 시뮬레이션된 디바이스는 실제 디바이스를 연결하기 전에 IoT Central 애플리케이션의 동작을 테스트하는 데 유용합니다.

이 빠른 시작에서는 ESP32-Azure IoT Kit 개발 보드에 대한 디바이스 템플릿을 추가하고 시뮬레이션된 디바이스를 만듭니다. 실제 디바이스가 없어도 이 빠른 시작을 완료할 수 있습니다. 디바이스 시뮬레이션을 사용할 것입니다. ESP32 디바이스는 다음과 같습니다.

* 원격 분석(예: 온도)을 보냅니다.
* 디바이스가 다시 부팅된 이후 최대 온도와 같은 디바이스 특정 속성을 보고합니다.
* 다시 부팅과 같은 명령에 응답합니다.
* 일반 디바이스 속성(예: 펌웨어 버전 및 일련 번호)을 보고합니다.

## <a name="prerequisites"></a>사전 요구 사항

[Azure IoT Central 애플리케이션 만들기](./quick-deploy-iot-central.md) 빠른 시작을 완료하여 **사용자 지정 앱 > 사용자 지정 애플리케이션** 템플릿으로 IoT Central 애플리케이션을 만듭니다.

## <a name="create-a-device-template"></a>디바이스 템플릿 만들기

새 디바이스 템플릿을 애플리케이션에 추가하려면 왼쪽 창에서 **디바이스 템플릿** 탭을 선택합니다.

:::image type="content" source="media/quick-create-simulated-device/device-definitions.png" alt-text="비어 있는 디바이스 템플릿 목록을 보여 주는 스크린샷":::

디바이스 템플릿에는 다음을 정의하는 디바이스 모델이 포함됩니다.

* 디바이스에서 보내는 원격 분석.
* 디바이스 속성
* 디바이스에서 응답하는 명령

### <a name="add-a-device-template"></a>디바이스 템플릿 추가하기

디바이스 모델을 IoT Central 애플리케이션에 추가하기 위한 몇 가지 옵션이 있습니다. 모델을 처음부터 새로 만들어도 되고, 파일에서 모델을 가져와도 되고, 디바이스 카탈로그에서 디바이스를 선택할 수도 있습니다. 또한 IoT Central은 실제 디바이스가 처음 연결될 때 애플리케이션에서 자동으로 리포지토리로부터 모델을 가져오는 *디바이스 우선* 방법을 지원합니다.

이 빠른 시작에서는 디바이스 카탈로그에서 디바이스를 선택하여 디바이스 템플릿을 만듭니다.

다음 단계에서는 디바이스 카탈로그를 사용하여 **ESP32** 디바이스에 대한 모델을 가져오는 방법을 보여 줍니다. 이러한 디바이스는 온도와 같은 원격 분석을 애플리케이션에 보냅니다.

1. 새 디바이스 템플릿을 추가하려면 **디바이스 템플릿** 페이지에서 **+ 새로 만들기** 를 선택합니다.

1. **유형 선택** 페이지의 **미리 구성된 디바이스 템플릿 사용** 섹션에서 **ESP32-Azure IoT Kit** 타일을 찾을 때까지 아래로 스크롤합니다.

1. **ESP32-Azure IoT Kit** 타일, **다음: 검토** 를 차례로 선택합니다.

1. **검토** 페이지에서 **만들기** 를 선택합니다.

1. 몇 초 후 새 디바이스 템플릿을 볼 수 있습니다.

    :::image type="content" source="media/quick-create-simulated-device/devkit-template.png" alt-text="ESP32 디바이스에 대한 디바이스 템플릿을 보여 주는 스크린샷":::

    템플릿 이름은 **센서 조절기** 입니다. 모델에는 **센서 조절기**, **SensorTemp** 및 **디바이스 정보 인터페이스** 와 같은 구성 요소가 포함되어 있습니다. 구성 요소는 ESP32 디바이스의 기능을 정의합니다. 기능에는 원격 분석, 속성 및 명령이 포함됩니다.

### <a name="add-cloud-properties"></a>클라우드 속성 추가

디바이스 템플릿에는 클라우드 속성이 포함될 수 있습니다. 클라우드 속성은 IoT Central 애플리케이션에만 있으며 디바이스에서 보내거나 받지 않습니다. 두 개의 클라우드 속성을 추가하려면 다음을 수행합니다.

1. **클라우드 속성**, **+ 클라우드 속성 추가** 를 차례로 선택합니다. 다음 표의 정보를 사용하여 디바이스 템플릿에 두 가지 클라우드 속성을 추가합니다.

    | 표시 이름      | 의미 체계 유형 | 스키마 |
    | ----------------- | ------------- | ------ |
    | 마지막 서비스 날짜 | None          | Date   |
    | 고객 이름     | None          | String |

1. **저장** 을 선택하여 변경 내용을 저장합니다.

    :::image type="content" source="media/quick-create-simulated-device/cloud-properties.png" alt-text="두 클라우드 속성을 보여 주는 스크린샷":::

## <a name="views"></a>보기

디바이스와 관련된 정보를 표시하도록 애플리케이션을 사용자 지정할 수 있습니다. 사용자 지정을 통해 다른 사용자가 애플리케이션에 연결된 디바이스를 관리할 수 있습니다. 디바이스와 상호 작용할 두 가지 유형의 보기를 만들 수 있습니다.

* 디바이스 및 클라우드 속성을 보고 편집하는 양식
* 디바이스에서 보내는 원격 분석을 포함하여 디바이스를 시각화하는 대시보드

### <a name="default-views"></a>기본 보기

기본 보기는 중요한 디바이스 정보를 빠르게 시각화할 수 있는 방법입니다. 디바이스 템플릿에 대해 최대 세 개의 기본 보기를 생성할 수 있습니다.

* **명령** 보기를 사용하면 디바이스에 명령을 보낼 수 있습니다.
* **개요** 보기는 차트와 메트릭을 사용하여 디바이스 원격 분석을 표시합니다.
* **정보** 보기는 디바이스 속성을 표시합니다.

디바이스 템플릿에서 **보기** 노드를 선택합니다. 템플릿을 추가할 때 IoT Central이 **개요**, **정보** 및 **원시 데이터** 보기를 생성했음을 알 수 있습니다.

디바이스를 관리하기 위해 새 양식을 추가하려면 다음을 수행합니다.

1. **보기** 노드를 선택하고 **디바이스 및 클라우드 데이터 편집** 타일을 선택하여 새 보기를 추가합니다.

1. 양식 이름을 **디바이스 관리** 로 변경합니다.

1. **고객 이름** 및 **마지막 서비스 날짜** 클라우드 속성과 **대상 온도** 속성을 선택합니다. 그런 다음, **섹션 추가** 를 선택합니다.

    :::image type="content" source="media/quick-create-simulated-device/new-form.png" alt-text="디바이스 템플릿에 추가된 새 양식을 보여 주는 스크린샷":::

1. **저장** 을 선택하여 새 양식을 저장합니다.

## <a name="publish-device-template"></a>디바이스 템플릿 게시

시뮬레이션된 디바이스를 만들거나 실제 디바이스를 연결하려면 디바이스 템플릿을 게시해야 합니다. 템플릿을 처음 만들 때 IoT Central에서 템플릿을 게시했지만, 업데이트된 버전을 게시해야 합니다.

디바이스 템플릿을 게시하려면 다음을 수행합니다.

1. **디바이스 템플릿** 페이지에서 **센서 조절기** 디바이스 템플릿으로 이동합니다.

1. 페이지 상단에 있는 명령 모음에서 **게시** 를 선택합니다.

1. 표시되는 대화 상자에서 **게시** 를 선택합니다.

디바이스 템플릿을 게시하면 **디바이스** 페이지에 표시됩니다. 게시된 디바이스 템플릿에서 디바이스 모델을 편집하려면 먼저 새 버전을 만들어야 합니다. 그러나 클라우드 속성, 사용자 지정 및 보기는 버전 관리를 수행하지 않고도 게시된 디바이스 템플릿에서 수정할 수 있습니다. 변경한 후 **게시** 를 선택하여 실제 디바이스 및 시뮬레이션된 디바이스에 사용할 변경 내용을 푸시합니다.

## <a name="add-a-simulated-device"></a>시뮬레이트된 디바이스 추가

시뮬레이션된 디바이스를 애플리케이션에 추가하려면 사용자가 만든 **ESP32** 디바이스 템플릿을 사용합니다.

1. 새 디바이스를 추가하려면 왼쪽 창에서 **디바이스** 를 선택합니다. **디바이스** 탭에는 ESP32 디바이스에 대한 **모든 디바이스** 및 **센서 조절기** 디바이스 템플릿이 표시됩니다. **센서 조절기** 를 선택합니다.

1. 시뮬레이션된 DevKit 디바이스를 추가하려면 **+ 새로 만들기** 를 선택합니다. 제안된 **디바이스 ID** 를 사용하거나 직접 입력합니다. 디바이스 ID는 문자, 숫자 및 `-` 문자를 포함할 수 있습니다. 새 디바이스의 이름을 입력할 수도 있습니다. **이 디바이스를 시뮬레이션함** 이 **예** 로 설정되어 있는지 확인한 다음, **만들기** 를 선택합니다.

    :::image type="content" source="media/quick-create-simulated-device/simulated-device.png" alt-text="시뮬레이션된 센서 조절기 디바이스를 보여 주는 스크린샷":::

이제 시뮬레이션된 데이터를 사용하여 이전에 만든 보기와 상호 작용할 수 있습니다.

1. **디바이스** 페이지에서 시뮬레이션된 디바이스를 선택합니다.

    * **개요** 보기에는 시뮬레이션된 원격 분석 플롯이 표시됩니다.

        :::image type="content" source="media/quick-create-simulated-device/simulated-telemetry.png" alt-text="시뮬레이션된 디바이스에 대한 개요 페이지를 보여 주는 스크린샷":::

    * **정보** 보기에 속성 값이 표시됩니다.

    * **명령** 보기를 사용하면 디바이스에서 **다시 부팅** 과 같은 명령을 실행할 수 있습니다.

    * **디바이스 관리** 보기는 디바이스를 관리하기 위해 만든 양식입니다.

    * **원시 데이터** 보기를 통해 디바이스에서 보낸 원시 원격 분석 및 속성 값을 볼 수 있습니다. 이 보기는 디바이스를 디버깅하는 데 유용합니다.

## <a name="next-steps"></a>다음 단계

이 빠른 시작에서는 ESP32 디바이스에 대한 **센서 컨트롤러** 디바이스 템플릿을 만들고, 시뮬레이션된 디바이스를 애플리케이션에 추가하는 방법을 알아보았습니다.

애플리케이션에 연결된 디바이스를 모니터링하는 방법에 대해 자세히 알아보려면 빠른 시작을 계속 진행합니다.
