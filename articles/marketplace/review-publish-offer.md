# <a name="review-and-publish-an-offer-to-the-commercial-marketplace"></a>상용 marketplace에 제품 검토 및 게시

이 문서에서는 파트너 센터를 사용 하 여 미리 보기에 제품을 제출 하 고 제품을 미리 본 다음 Microsoft 상용 marketplace에 게시 하는 방법을 보여 줍니다. 게시 단계를 진행 하면서 게시 상태를 확인 하는 방법에 대해서도 설명 합니다. 게시 하려는 제품이 이미 만들어져 있어야 합니다.

## <a name="go-to-your-offer-in-the-commercial-marketplace"></a>상업적 marketplace의 제품으로 이동

1. [파트너 센터](https://partner.microsoft.com/dashboard/home)에 로그인합니다.
2. 왼쪽 탐색 메뉴에서 **상업용 마켓플레이스**  >  **개요**를 선택 합니다.
3. **개요** 탭의 **제안**에서 다음 상태 표시기 중 하나가 각 제안의 **상태** 열에 표시 됩니다.

    | 상태 | Description |
    | ------------ | ------------- |
    | Draft | 제안이 만들어졌지만 게시 되 고 있지 않습니다. |
    | 게시 진행 중 | 제안은 게시 프로세스를 통해 작동 합니다. |
    | 주의가 필요함 | 인증 하는 동안 이나 다른 게시 단계 중에 중요 한 문제가 발견 되었습니다. |
    | 미리 보기 | 이제 게시자가 최종 확인을 기다립니다 제품을 인증 했습니다. **라이브 전환** 을 선택 하 여 제품을 라이브 게시 합니다. |
    | 라이브 | 제품은 marketplace에 살고 있으며 고객이 확인 하 고 얻을 수 있습니다. |
    | 보류 중인 판매 중지 | 게시자가 제품 또는 계획에서 "판매 중지"를 선택 했지만 작업이 아직 완료 되지 않았습니다. |
    | Marketplace에서 사용할 수 없음 | 마켓플레이스에 이전에 게시된 제품이 제거되었습니다. |
    |||

4. **제품 별칭** 열에서 미리 보고 게시할 제품을 선택 합니다.

## <a name="submit-your-offer-to-preview"></a>제품을 제출 하 여 미리 보기

1. 제품을 미리 보기에 제출 하려면 포털의 오른쪽 위 모서리에서 **검토 및 게시** 를 선택 합니다. **검토 및 게시** 페이지가 나타납니다.
1. 각 페이지의 **상태** 열에 **완료**가 표시 되는지 확인 합니다. 가능한 세 가지 상태는 다음과 같습니다.

   - **시작 되지 않음** -페이지가 완전 하지 않습니다.
   - **불완전** -페이지에 필요한 정보가 없거나 수정 해야 하는 오류가 있습니다. 페이지로 돌아가서 업데이트 해야 합니다.
   - **완료** – 페이지가 완료 되었습니다. 필요한 모든 데이터가 제공 되었으며 오류가 없습니다.

1. 페이지의 상태가 **완료**가 아닌 경우 **페이지** 열에서 페이지 이름을 선택 하 고 문제를 수정한 다음 페이지를 저장 하 고 **검토 및 게시** 를 다시 선택 하 여이 페이지로 돌아갑니다.
1. 모든 페이지가 완료 되 면 **인증에 대 한 정보** 상자에서 인증 팀에 테스트 지침을 제공 하 여 앱이 제대로 테스트 되었는지 확인 합니다. 앱을 이해 하는 데 도움이 되는 보조 노트를 제공 합니다.
1. 게시하기 위해 제품을 제출하려면 **게시**를 선택합니다. **제품 개요** 페이지가 표시 되 고 게시 상태가 표시 됩니다.

## <a name="validation-and-publishing-steps"></a>유효성 검사 및 게시 단계

**게시**를 선택 하면 유효성 검사 및 게시 프로세스가 순서 대로 진행 됩니다. 가장 일반적인 게시 프로세스가 다음 표에 나와 있습니다.

| 단계 | 수행되는 작업 |
| ------------ | ------------- | ------------- |
| 자동 유효성 검사 | 자동 유효성 검사 집합을 처리 합니다. |
| 인증 | 수동 유효성 검사를 수행 합니다. |
| 미리 보기 만들기 | 제품 미리 보기에 대 한 목록 페이지는 미리 보기 링크를 가진 모든 사용자가 사용할 수 있습니다. Microsoft (불가능)를 통해 제품을 판매 하는 경우 제품의 **대상 미리 보기** 페이지에서 지정한 사용자만이 제품을 구매 하 고 테스트에 액세스할 수 있습니다. |
| 게시자 승인 | 제품을 미리 보고 승인할 수 있는 요청이 포함 된 전자 메일을 보냅니다. |
| 게시 | 일련의 단계를 실행 하 여 미리 보기 제품이 상용 marketplace에 라이브 게시 되었는지 확인 합니다. |
|||

## <a name="automated-validation-phase"></a>자동 유효성 검사 단계

게시 프로세스의 첫 번째 단계는 자동 유효성 검사 집합입니다. 각 유효성 검사 단계는 제품을 만들 때 선택한 기능에 해당 합니다. 제품을 게시 프로세스의 다음 단계로 진행 하기 전에 각 유효성 검사를 완료 해야 합니다.

- **구매 흐름 설정 제공** (<10 분)

   Azure Portal를 통해 고객이 구매할 때 제품이 충족 될 수 있는지 확인 합니다. 이 단계는 Microsoft를 통해 판매 되는 제품에만 적용 됩니다.

- **테스트 드라이브 데이터 유효성 검사** (~ 5 분)

   제품의 기술 구성 페이지에서 제공한 데이터의 유효성을 검사 합니다. Test drive 기능을 테스트 하 고 승인 합니다. 이 단계는 테스트 드라이브를 사용 하도록 설정한 경우에만 적용할 수 있습니다.

- **테스트 드라이브 프로 비전** (~ 30 분)

    이전 단계에서 테스트 드라이브의 데이터 및 기능에 대 한 유효성을 검사 한 후에는 고객이 사용할 수 있도록 테스트 드라이브의 인스턴스를 배포 하 고 복제 합니다. 이 단계는 테스트 드라이브를 사용 하도록 설정한 경우에만 적용할 수 있습니다.

- **리드 관리 유효성 검사 및 등록** (15 분 <)

    고객 관리 시스템에서 **제품 설치** 페이지에 제공 된 세부 정보에 따라 고객 리드를 받을 수 있음을 확인 합니다. 이 단계는 잠재 고객 관리를 사용 하도록 설정한 경우에만 적용 됩니다.

## <a name="certification-phase"></a>인증 단계

상업적 marketplace에 제출 된 제안은 게시 되기 전에 인증 되어야 합니다. 에서 제공 하는 엄격한 테스트를 제공 합니다. 자세히 알아보려면 [상업적 marketplace 인증 정책](https://aka.ms/commercial-marketplace-certification-policies)을 참조 하세요.

### <a name="types-of-validation-that-take-place-during-certification"></a>인증 하는 동안 발생 하는 유효성 검사 유형

제출 된 각 제품에 대 한 인증 프로세스에는 세 가지 수준의 유효성 검사가 포함 되어 있습니다.

- 게시자 비즈니스 자격
- 콘텐츠 유효성 검사
- 기술 유효성 검사

#### <a name="publisher-business-eligibility"></a>게시자 비즈니스 자격

각 제품 유형은 필요한 기본 자격 조건의 집합을 검사 합니다. 이 조건에는 게시자 MPN 상태, 역량 보유, 역량 수준 등이 포함 될 수 있습니다.

#### <a name="content-validation"></a>콘텐츠 유효성 검사

제품을 만들 때 입력 한 정보는 품질 및 관련성이 있는지 확인 됩니다. 이러한 검사는 세부 정보, 가격 책정, 가용성, 관련 계획 등을 나열 하는 marketplace의 항목을 검토 합니다. Azure Marketplace 및 Microsoft AppSource의 목록 기준을 충족 하기 위해 제품에 다음과 같은 내용이 포함 되어 있는지 확인 합니다.

- 제품을 정확 하 게 설명 하는 제목
- 철저 한 개요 및 가치 제안을 제공 하는 잘 작성 된 설명
- 품질 스크린샷 및 비디오
- 제품이 Microsoft 플랫폼 및 도구를 활용 하는 방법에 대 한 설명입니다.

[일반 목록 정책을](https://aka.ms/commercial-marketplace-certification-policies#100-general)읽어 콘텐츠 유효성 검사 조건에 대해 자세히 알아보세요.

#### <a name="technical-validation"></a>기술 유효성 검사

기술 유효성을 검사 하는 동안 제품 (패키지 또는 이진)은 다음 검사를 수행 합니다.

- 맬웨어 검색 됨
- 모니터링 된 네트워크 호출
- 패키지 분석 됨
- 제품 기능에 대 한 철저 한 검사

제품은 강력한 기능을 보장 하기 위해 다양 한 플랫폼 및 버전에서 테스트 됩니다.

### <a name="certification-failure-report"></a>인증 실패 보고서

제품이 목록, 기술 또는 정책 검사에 실패 하거나 해당 유형의 제품을 제출할 수 없는 경우 사용자에 게 인증 실패 보고서를 전자 메일로 보냅니다.

이 보고서에는 검토 메모와 함께 실패 한 모든 정책에 대 한 설명이 포함 되어 있습니다. 이 전자 메일 보고서를 검토 하 고, 문제를 해결 하 고, 필요한 경우 제품을 업데이트 하 고, 파트너 센터에서 [상용 마켓플레이스 포털](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) 을 사용 하 여 제품을 다시 제출 합니다. 인증을 통과할 때까지 필요한 만큼 제품을 다시 제출할 수 있습니다.

## <a name="preview-creation-phase"></a>미리 보기 생성 단계

미리 보기 생성 단계 중에 제품의 사용자 **미리 보기** 페이지 (있는 경우)에서 지정한 대상 그룹에만 액세스할 수 있는 제품의 버전을 만듭니다. 제품의 미리 보기 버전은 제품을 라이브로 게시할 때까지 미리 보기 대상 외부의 사용자가 사용할 수 없습니다.

> [!NOTE]
> 미리 보기 대상 그룹을 사용 하 여 조직에서 제공 하는 제품에 대 한 정보를 제공 하지 마세요. 대신 개인 제품 옵션을 사용 하세요. 이 시점에서 제품이 완전히 테스트 및 검증 되지 않았으며 외부 배포를 위한 준비가 되지 않았습니다.

## <a name="publisher-sign-off-phase"></a>게시자 로그 오프 단계

제품을 검토 하 고 로그 오프할 준비가 되 면 제품 미리 보기를 검토 하 고 승인 하도록 요청 하는 전자 메일을 보냅니다. 브라우저에서 **제품 개요** 페이지를 새로 고쳐 제품이 게시자 로그 오프 단계에 도달 했는지 확인할 수도 있습니다. 있는 경우 **라이브 이동** 단추와 미리 보기 링크를 사용할 수 있습니다.

다음 스크린샷은 SaaS 제품에 대 한 **제품 개요** 페이지를 보여 줍니다. 이 페이지에 표시 되는 유효성 검사 단계는 제품을 만들 때 선택한 내용 및 제안 유형에 따라 달라 집니다.

![파트너 센터의 제품에 대 한 제품 개요 페이지를 보여 줍니다. 라이브 이동 단추와 미리 보기 링크가 표시 됩니다.](./partner-center-portal/media/publish-status-publisher-signoff.png)

### <a name="to-preview-your-offer-and-sign-off"></a>제품을 미리 보고 로그 오프 하려면

1. **제품 개요** 페이지에서 제품을 미리 보려면 **라이브 이동** 단추 아래의 링크를 선택 합니다.

   > [!NOTE]
   > 제품을 만들 때 선택한 옵션에 따라 AppSource 미리 보기, Azure Marketplace 미리 보기 또는 둘 다에 대 한 링크가 있습니다. Microsoft를 통해 제품을 판매 하도록 선택한 경우 미리 보기 대상에 추가 된 모든 사용자가 제품의 획득 및 배포를 테스트 하 여이 단계에서 요구 사항을 충족 하는지 확인할 수 있습니다.

1. 제품을 미리 본 후에 변경 하려는 경우 편집 하 고 다시 제출 하 여 새 미리 보기를 게시할 수 있습니다. 자세한 내용은 [상용 marketplace에서 기존 제품 업데이트](./partner-center-portal/update-existing-offer.md)를 참조 하세요.

1. 미리 보기를 승인한 후 제품을 상용 marketplace에 게시 하려면 **라이브 전환**을 선택 합니다.

   > [!TIP]
   > 제품이 이미 라이브 이며 marketplace에서 공개에 사용할 수 있는 경우 **Live go**를 선택할 때까지 모든 업데이트는 라이브 상태로 전환 되지 않습니다.

## <a name="publish-phase"></a>게시 단계

이제 제품을 사용 하도록 선택 하 여 상업적 marketplace에서 사용할 수 있게 되었으므로 live 제품이 제품 미리 보기 버전 처럼 구성 되도록 일련의 최종 유효성 검사를 수행 합니다.

- **구매 흐름 설정 제공** (>10 분)

    Azure Portal를 통해 고객이 구매할 때 제품이 충족 될 수 있는지 확인 합니다. 이 단계는 Microsoft를 통해 판매 되는 제품에만 적용 됩니다.

- **테스트 드라이브 데이터 유효성 검사** (~ 5 분)

    제품의 기술 구성 페이지에서 제공한 데이터의 유효성을 검사 합니다. Test drive 기능을 테스트 하 고 승인 합니다. 이 단계는 테스트 드라이브를 사용 하도록 설정한 경우에만 적용할 수 있습니다.

- **테스트 드라이브 프로 비전** (~ 30 분)

    사용자가 사용할 수 있도록 테스트 드라이브의 인스턴스를 배포 하 고 복제 합니다. 이 단계는 테스트 드라이브를 사용 하도록 설정한 경우에만 적용할 수 있습니다.

- **리드 관리 유효성 검사 및 등록** (15 분 >)

    고객 관리 시스템에서 **제품 설치** 페이지에 제공 된 세부 정보에 따라 고객 리드를 받을 수 있음을 확인 합니다. 이 단계는 잠재 고객 관리를 사용 하도록 설정한 경우에만 적용 됩니다.

- **최종 게시 (>30 분)**

    제품이 marketplace에서 공개적으로 제공 되는 것을 보장 합니다.

이러한 유효성 검사를 완료 한 후에는 제품이 marketplace에 살고 있습니다.

### <a name="reporting-usage-of-preview-and-other-internal-usage"></a>미리 보기 및 기타 내부 사용 현황 보고

종단 간 구매 및 설정 흐름의 유효성을 검사 하려면 미리 보기로 제공 되는 동안 제품을 구매 하세요. 먼저 Microsoft에 [지원 티켓](https://aka.ms/marketplacesupport) 을 알리고 요금을 처리 하지 않도록 합니다.
